# 仓库上下文

## 仓库目的

这个仓库是一个以学习为优先的 Nuxt 项目，面向已经会 Vue、希望进一步掌握 Nuxt、SSR、前后端一体化模式以及 Agent 协作方式的前端开发者。

代码规模应保持在可以整体读懂的范围内，但又要足够完整，能展示真实的 Nuxt 开发方式。

## 核心学习目标

- 理解 Nuxt 的目录约定：`pages/`、`layouts/`、`components/`、`middleware/`、`server/api/`
- 理解路由参数、query 参数、动态路由和嵌套路由
- 理解 `useFetch()` 和 `$fetch()` 的区别
- 理解基于 cookie 的 SSR 登录态，以及 `useRequestHeaders()` 的作用
- 理解 `useState()` 如何在 Nuxt 中做共享状态
- 理解 Vite 在 Nuxt 中的位置，以及 `nuxt.config.ts` 的职责
- 为后续学习 Agent 开发打下服务端和工作流基础

## 术语表

### 学习页面

以讲清一个 Nuxt 概念为主要目标的页面，通常配合注释和可运行示例。

### 跟敲练习

要求学习者自己动手输入、运行和修改的小任务，而不是只读不练。

### Middleware

在进入页面前执行的路由守卫逻辑。本仓库里它主要控制页面访问和重定向，但它不是最终安全边界。

### Server API

位于 `server/api/` 下的 Nuxt 服务端处理函数。它才是鉴权和权限控制的真实执行点。

### 登录会话

基于 cookie 的登录状态，会在浏览器、middleware 和服务端接口之间传递。

### 共享登录状态

通过 `useState('auth:user', ...)` 创建，并通过 `useAuth()` 读取和更新的前端共享状态。

## 架构边界

- `pages/`：负责路由级页面内容
- `layouts/`：负责公共页面框架
- `components/`：负责可复用 UI 组件
- `composables/`：负责可复用状态逻辑和组合式函数
- `middleware/`：负责进入页面前的检查逻辑
- `server/api/`：负责服务端行为和真实权限判断

## 代码约束

- 优先使用清晰直接的示例，不要过度抽象
- 注释重点解释 Nuxt 特性，而不是重复基础 Vue 语法
- 示例尽量本地可运行，必要时允许使用本地 mock 数据
- 权限相关示例必须明确区分“前端页面拦截”和“后端接口鉴权”

## 非目标

- 不把这个仓库扩展成通用企业脚手架
- 不用大型工具层把学习重点包起来
- 不为了扩展性牺牲可读性
